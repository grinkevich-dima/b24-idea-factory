# Идея 01 — CRM: SLA реакции на лид — скрины, безопасность, FAQ

## 1) План скриншотов для Маркета (6–8)
1. **Главный дашборд SLA**\n
   - карточки: “Просрочено”, “В зоне риска”, “Ок”, “Среднее время реакции”\n
   - график по дням (простая линия)\n
   - фильтры: период, источник, ответственный.
2. **Список просроченных лидов**\n
   - колонки: лид, источник, ответственный, время до касания/просрочка, текущий статус\n
   - быстрый переход в карточку лида.
3. **Настройка SLA порогов**\n
   - порог (мин/часы)\n
   - рабочие часы (упрощённо)\n
   - исключения (тихие часы/выходные).
4. **Выбор “первого касания” (настройка)**\n
   - режим A: первая смена статуса\n
   - режим B: первое дело/активность (если включено)\n
   - подсказка “как это влияет на расчёт”.
5. **Эскалации и уведомления**\n
   - правила: кому, когда, как часто\n
   - пример дайджеста (превью).
6. **KPI по менеджерам**\n
   - таблица: среднее, P90, % в SLA, кол-во просрочек\n
   - сортировка по “просрочкам”.
7. (Опционально) **Сравнение источников**\n
   - какая реклама/канал даёт лиды, которые “не успевают обработать”.

## 2) Чек‑лист модерации и безопасности
### Права (минимальный подход)
- **CRM**: чтение лидов + чтение базовых полей (без необходимости изменения).\n
  Если для MVP где-то требуется запись (например, сохранить метрику в поле) — лучше хранить в своём хранилище/настройках приложения.
- **Пользователи (минимальный)**: только для отображения “ответственный/руководитель” в фильтрах.
- **Чат и уведомления**: только отправка уведомлений/дайджестов (не читать содержимое).

### Что мы НЕ делаем (важно явно написать)
- Не читаем и не анализируем **тексты сообщений**, письма, содержимое звонков.\n
- Не передаём данные CRM на сторонние сервисы.

### Хранение данных
- Храним только **метрики времени** (timestamp создания, timestamp первого касания, SLA‑порог, статус расчёта).\n
- Период хранения: ограничиваем (например, последние 90–180 дней) или по настройке.

### Производительность
- По умолчанию считаем по ограниченному периоду и активным лидам.\n
- Агрегаты кэшируем; пересчёт инкрементальный при изменениях.

## 3) FAQ для Маркета (готовый текст)
**Какие данные читает приложение?**\n
Только данные CRM, необходимые для расчёта времени реакции: дата создания лида, ответственный, статус/стадия и (при включении опции) факт первого дела/активности — без чтения содержимого коммуникаций.

**Читает ли приложение тексты сообщений/писем?**\n
Нет. Приложение считает только временные метрики и статусы.

**Можно ли ограничить видимость по правам?**\n
Да, приложение работает в рамках прав доступа Bitrix24: пользователь видит только доступные ему лиды.

**Сколько времени нужно, чтобы получить пользу?**\n
Первый отчёт доступен сразу после установки и выбора порогов SLA.
