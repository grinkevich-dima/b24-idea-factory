# 06 — Скидки под контролем: скриншоты + безопасность + FAQ

## План скриншотов (6–8)

1. **Главный экран — Журнал нарушений**: таблица с колонками (дата, менеджер, сделка, товар, порог %, факт %, превышение). Красные строки = критичные превышения. Фильтры сверху.
2. **Настройка правил**: форма создания правила — выбор воронки, стадии, макс. скидка %, получатель алерта. Список существующих правил ниже.
3. **Уведомление в чате**: скриншот чата Bitrix24 с сообщением от бота: «⚠️ Превышение скидки: [Менеджер] дал 25% (порог 15%) на сделку [Название]. [Ссылка]».
4. **Карточка нарушения**: детальная информация — сделка, товары, примененные скидки vs пороги, история нарушений этого менеджера.
5. **Дашборд (обзор)**: 3 числа сверху: «Нарушений сегодня: 3», «За неделю: 12», «Потенциальные потери: 45 000 ₽». Ниже — график нарушений по дням.
6. **Пустое состояние**: экран при 0 нарушениях — «Всё под контролем ✅ Нарушений нет. Ваши правила работают.»
7. **Мобильная версия**: уведомление на телефоне (push) с кратким текстом о нарушении.

## Стратегия минимальных прав

| Право | Обоснование | Уровень |
|-------|-------------|---------|
| crm.item.read (сделки) | Чтение товарных позиций и скидок | Чтение |
| crm.product.read | Чтение каталога для сопоставления | Чтение |
| im.message.add | Отправка уведомлений о нарушении | Запись (только сообщения) |
| user.basic | Определение роли/должности менеджера | Чтение |

**Не запрашиваем:**
- Запись/изменение сделок
- Доступ к контактам/компаниям
- Полный доступ к чатам
- Доступ к лидам/счетам

## Что приложение НЕ читает / НЕ хранит

- ❌ Содержание переписок и комментариев в сделках
- ❌ Персональные данные клиентов (телефоны, email, адреса)
- ❌ Финансовые документы и счета
- ❌ Историю звонков и их записи
- ❌ Данные других модулей (задачи, календарь, диск)

## Политика хранения

- Храним: только агрегаты (дата нарушения, ID сделки, ID менеджера, порог, факт, разница)
- Срок хранения: 90 дней (настраивается), затем автоудаление
- Не храним: содержимое сделок, персональные данные, контент товаров
- Данные хранятся на серверах в РФ (если требуется)

## Производительность

- Проверка срабатывает только при событии изменения товарных позиций сделки (не периодически)
- Среднее время проверки: < 200 мс (1 API-запрос)
- Нет фоновой нагрузки на портал между событиями
- Журнал: пагинация по 50 записей

## FAQ (текст для Маркета)

**Вопрос: Приложение блокирует сохранение сделки?**
Ответ: Нет. Приложение фиксирует нарушение и уведомляет руководителя, но не мешает работе менеджера. Блокировка — в планах на будущие версии (по запросу).

**Вопрос: Какие данные приложение хранит?**
Ответ: Только факт нарушения: дата, ID сделки, ID менеджера, размер скидки. Не храним персональные данные клиентов, переписки и финансовые документы.

**Вопрос: Приложение замедляет работу портала?**
Ответ: Нет. Проверка занимает < 200 мс и срабатывает только при изменении товаров в сделке. Фоновой нагрузки нет.

**Вопрос: Можно ли настроить разные пороги для разных менеджеров?**
Ответ: Да. Правила настраиваются по воронке, стадии и роли/должности сотрудника.

**Вопрос: Поддерживается ли скидка в рублях (не только в %)?**
Ответ: В текущей версии — только процентная скидка. Скидка в сумме — в ближайшем обновлении.

**Вопрос: Как удалить данные приложения?**
Ответ: При удалении приложения все данные журнала удаляются автоматически. Также можно очистить журнал вручную из настроек.
