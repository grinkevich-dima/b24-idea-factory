# 07 — Зависшие задачи без изменений: скриншоты + безопасность + FAQ

## План скриншотов (6–8)

1. **Главный экран — Список зависших задач**: таблица (название задачи, проект, ответственный, дней молчания, последнее изменение). Сортировка по «дням молчания» desc. Цветовая индикация: жёлтый (5–7 дней), оранжевый (7–14), красный (14+).
2. **Дайджест в чате**: скриншот сообщения в чате Bitrix24: «📋 Дайджест зависших задач (7 шт.):\n1. [Задача] — 12 дней без изменений (@Иванов)\n2. [Задача] — 9 дней...».
3. **Настройки**: форма — порог дней (слайдер 3–30), расписание дайджеста (ежедневно/еженедельно), получатели, исключённые проекты.
4. **Фильтры в действии**: список с примененными фильтрами — «Проект: Маркетинг», «Ответственный: Петров» — показывает только релевантные.
5. **Пустое состояние**: «🎉 Нет зависших задач! Все задачи активны.» — мотивирующий экран.
6. **Исключение задачи**: попап «Исключить из мониторинга? Задача не будет попадать в дайджест.» с кнопками Да/Нет.
7. **Статистика**: мини-дашборд — «Зависших сейчас: 12 | На прошлой неделе: 18 | Тренд: ↓ -33%».

## Стратегия минимальных прав

| Право | Обоснование | Уровень |
|-------|-------------|---------|
| task.item.read | Чтение задач (статус, дата изменения, ответственный) | Чтение |
| im.message.add | Отправка дайджеста в чат | Запись (только сообщения) |
| user.basic | Определение имени ответственного | Чтение |

**Не запрашиваем:**
- Запись/изменение задач
- Доступ к CRM
- Полный доступ к чатам
- Доступ к файлам/диску
- Доступ к комментариям задач (только факт наличия)

## Что приложение НЕ читает / НЕ хранит

- ❌ Содержание комментариев к задачам
- ❌ Вложенные файлы
- ❌ Описание задач (только заголовок для отображения)
- ❌ Данные CRM, звонков, переписок
- ❌ Персональные данные сотрудников (только имя + ID)

## Политика хранения

- Храним: ID задачи, дата последнего изменения (из API), флаг «исключена», дата обнаружения
- Не храним: содержимое задач, файлы, комментарии
- Кэш: обновляется каждый час, хранится 24 часа
- При удалении приложения: все данные удаляются

## Производительность

- Запрос к API: раз в час (cron) — выборка задач с фильтром `CHANGED_DATE < now - N дней`
- По умолчанию: только задачи за последние 90 дней (исключаем архивные)
- Пагинация: по 50 задач на странице (API-лимит)
- Лимит: максимум 500 задач в выдаче (топ по «дням молчания»)
- Дайджест: формируется из кэша, не дополнительные API-запросы

## FAQ (текст для Маркета)

**Вопрос: Что считается «изменением» задачи?**
Ответ: Любое: смена статуса, комментарий, вложение файла, изменение ответственного, изменение срока. Если хоть что-то произошло — задача не считается зависшей.

**Вопрос: При первом запуске показывает 500 зависших задач — что делать?**
Ответ: Настройте фильтр «только задачи за последние 90 дней» и исключите завершённые проекты. Старые задачи-«хвосты» можно массово закрыть или исключить.

**Вопрос: Приложение замедляет портал?**
Ответ: Нет. Запрос к API выполняется раз в час в фоне. Нагрузка минимальна. Не влияет на скорость работы пользователей.

**Вопрос: Можно ли получать дайджест только по конкретному проекту?**
Ответ: Да. В настройках можно выбрать проекты для мониторинга или исключить ненужные.

**Вопрос: Приложение может переназначить задачу?**
Ответ: Нет. Приложение только информирует. Оно не изменяет задачи — это осознанный выбор для минимальных прав и безопасности.

**Вопрос: Как удалить данные?**
Ответ: При удалении приложения все данные (кэш, настройки, исключения) удаляются автоматически.
