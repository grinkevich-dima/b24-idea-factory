# Идея 03 — CRM: “Сделки без касаний” — скрины, безопасность, FAQ

## 1) План скриншотов (6–8)
1. **Главный экран “Без касаний”**\n
   - карточки: “>7 дней”, “>14 дней”, “>30 дней”, сумма в риске.\n
   - фильтры: воронка, стадия, ответственный, период.
2. **Список “забытых” сделок**\n
   - сделка, стадия, ответственный, последняя активность, сумма, “риск”.\n
   - сортировка по риску/сумме.
3. **Режим “Фокус” (TOP‑10)**\n
   - самые важные сделки + быстрые кнопки действий.
4. **Быстрые действия**\n
   - создать задачу follow‑up, поставить напоминание, открыть сделку.\n
   - шаблоны задач (1–2).
5. **Настройка “что считать активностью”**\n
   - переключатель: изменения сделки / дела‑активности / задачи.\n
   - исключения стадий.
6. **Эскалации/уведомления**\n
   - дайджест руководителю по менеджерам.

## 2) Чек‑лист модерации и безопасности
### Права (минимально)
- **CRM**: чтение сделок/стадий/воронок.\n
- **Задачи (если включены быстрые действия)**: создание задач (минимальные права на создание).\n
- **Чат и уведомления**: отправка уведомлений (без чтения сообщений).\n
- **Пользователи (минимальный)**: отображение сотрудников.

### Хранение
- timestamp последней активности + тип активности (без содержания).\n
- настройки периодов “тишины” и исключений.

## 3) FAQ (готовый текст)
**Что считается “касанием”?**\n
По умолчанию — изменения в сделке (например смена стадии/обновление). Можно переключить метод в настройках.

**Можно ли работать без задач?**\n
Да. В MVP достаточно списка и уведомлений. Быстрые действия (создание задач) включаются опционально.

**Есть ли риск спама?**\n
Нет, есть дайджест и лимиты уведомлений.
