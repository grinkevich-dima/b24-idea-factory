# Идея 22 — Чаты: Чек‑лист качества ответа

## 1) Коротко (одна фраза)
Простой чек‑лист помогает руководителю оценивать качество ответов операторов по стандартам без тяжёлой аналитики и внешних сервисов.

## 2) Для кого (ICP)
- Руководитель поддержки
- Руководитель продаж в чатах
- Тренер операторов

## 3) Боль и JTBD
- Качество ответов оценивается нерегулярно и субъективно.
- Нет компактного отчёта по типовым нарушениям стандарта коммуникации.

## 4) Измеримый эффект
- Рост доли ответов, соответствующих стандарту.
- Сокращение повторных ошибок операторов.

## 5) УТП (позиционирование)
**“Дайте руководителю быстрый чек‑лист оценки чатов и отчёт по слабым местам команды.”**

## 6) MVP (3–5 фич)
1. **Настраиваемые чек‑листы оценки.**
2. **Форма оценки выбранного диалога руководителем.**
3. **Отчёт по операторам и критериям.**
4. **Дайджест типовых нарушений.**

## 7) Реализуемость и тех‑чеклист (10 минут)
### Данные и объекты
- Чаты/диалоги, операторы, оценки, критерии чек‑листа.

### Триггеры и обновление
- Оценка запускается вручную из списка диалогов или отдельной панели.

### Права и безопасность
- Храним оценки и критерии; контент сообщений не копируем в хранилище приложения.

### Производительность
- Работаем с выбранными диалогами и ограниченным периодом оценки.

## 8) Риски (и как снизить)
- **Субъективность оценки** → единые критерии и подсказки.
- **Сопротивление операторов** → отчёт для обучения, не для наказаний в MVP.
- **Права на чаты** → минимизировать и прозрачно описать.

## 9) Метрики успеха
- Средний балл качества.
- Количество оценённых диалогов.
- Топ нарушенных критериев.

## 10) Поиск конкурентов (ключевые запросы для Маркета)
- “качество ответа”
- “чек лист оператора”
- “оценка чатов”
- “контроль поддержки”
- “Bitrix24 чат”

## 11) Упаковка для Маркета (черновик)
### Название
Чек‑лист качества ответов в чатах

### Буллеты ценности
- Единые критерии качества
- Оценка выбранных диалогов
- Отчёт по операторам
- Фокус на обучении команды
