# 08 — Просрочки по дедлайнам: радар — анализ конкурентов

## Методология поиска

Ключевые запросы: «просрочки задачи», «дедлайны», «контроль сроков», «deadline», «overdue tasks», «радар просрочек», «дедлайны проектов», «сроки задач», «просроченные задачи отчёт», «task deadlines dashboard».

## Конкуренты

### 1. Встроенный режим «Сроки» в Задачах

| Параметр | Значение |
|----------|----------|
| Обещание | Показать дедлайны задач во временной шкале |
| Механика | Режим просмотра «Сроки» — линейная шкала с задачами |
| Installs/оценка | Встроено (бесплатно) |
| Запрашиваемые права | — |
| Пробелы | Нет зонирования (горит/завтра/неделя). Нет агрегации. Нет алертов. Только свои задачи или один проект за раз. |

### 2. Триггер «Просроченная задача» (встроенный)

| Параметр | Значение |
|----------|----------|
| Обещание | Перемещение задачи на стадию при просрочке |
| Механика | Триггер срабатывает, когда дедлайн прошёл → перемещение + (опционально) уведомление |
| Installs/оценка | Встроено (тариф Профессиональный+) |
| Запрашиваемые права | — |
| Пробелы | Работает только в проектах с Канбан. Реактивный (когда уже просрочено), нет «предупреждения». Нет общей картины по всем проектам. |

### 3. Дедлайны проектов (Marketplace)

| Параметр | Значение |
|----------|----------|
| Обещание | Визуализация дедлайнов проектов с подсветкой просроченных |
| Механика | Список проектов с индикаторами статуса дедлайнов |
| Installs/оценка | ~100+ установок |
| Запрашиваемые права | Чтение задач |
| Пробелы | Уровень проектов, а не задач. Нет зонирования по срочности. Нет детализации до конкретной задачи. |

### 4. Диаграмма Ганта (встроенная)

| Параметр | Значение |
|----------|----------|
| Обещание | Визуализация задач во времени с зависимостями |
| Механика | Классическая диаграмма Ганта в разделе Задачи |
| Installs/оценка | Встроено |
| Запрашиваемые права | — |
| Пробелы | Хороша для планирования, но не для контроля просрочек. Нет фильтра «только проблемные». Перегружена для ежедневного мониторинга. |

### 5. Контроль сроков по Сделкам

| Параметр | Значение |
|----------|----------|
| Обещание | Контроль сроков в CRM-сделках |
| Механика | SLA/таймеры по стадиям сделки |
| Installs/оценка | ~200+ установок |
| Запрашиваемые права | Доступ к CRM |
| Пробелы | Фокус на CRM, а не на задачах/проектах. Другая аудитория. |

### 6. Автоматическая эскалация (кастомные роботы)

| Параметр | Значение |
|----------|----------|
| Обещание | Уведомление руководителю при просрочке 24ч+ |
| Механика | Робот + условие «просрочка > X» → сообщение |
| Installs/оценка | Встроенный функционал (требует настройки) |
| Запрашиваемые права | — |
| Пробелы | Нужно настраивать для каждого проекта вручную. Нет единого дашборда. Нет зонирования. |

## Топ слабых мест рынка

1. **Нет «радарного» экрана** — никто не даёт единую картину «горит / завтра / эта неделя» по ВСЕМ проектам и задачам.
2. **Реактивность** — встроенные инструменты сообщают, когда уже просрочено, но не предупреждают заранее.
3. **Фрагментация** — нужно заходить в каждый проект отдельно, нет кросс-проектной агрегации.
4. **Информационный шум** — Ганта и режим «Сроки» показывают ВСЕ задачи, а не только проблемные.
5. **Ручная настройка** — эскалации нужно настраивать проект за проектом.

## Что мы делаем лучше

- **Радарный UX**: 3 зоны (горит сейчас / загорится завтра / эта неделя) — мгновенная оценка ситуации.
- **Кросс-проектная агрегация**: все задачи всех проектов на одном экране.
- **Превентивность**: показываем задачи, которые СКОРО просрочатся, а не только уже просроченные.
- **Фильтр только проблемных**: никакого шума — только задачи, требующие внимания.
- **Автоматические алерты**: настроил один раз — работает для всех проектов.
