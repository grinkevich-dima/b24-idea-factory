# 06 — Скидки под контролем

## Подготовка (Шаг 0)

**Для РОПов и коммерческих директоров, которые теряют маржу из-за бесконтрольных скидок менеджеров в сделках Bitrix24 CRM — приложение снижает потери маржи на 15–30% за счёт автоматических порогов и журнала отклонений.**

## Идея

| Поле | Значение |
|------|----------|
| ICP/роль покупателя | РОП, коммерческий директор, владелец бизнеса (B2B, оптовая/розничная торговля) |
| Боль | Менеджеры дают скидки «от себя» без согласования; руководитель узнаёт постфактум, маржа тает. Нет единого инструмента контроля в CRM. |
| Измеримый эффект | Снижение случаев превышения порога скидки на 80%+; рост средней маржинальности сделок на 15–30% |
| MVP-объём | 1–2 недели: правила порогов (по воронке/стадии/роли), журнал отклонений, алерт руководителю |
| Основные риски | Разные модели скидок у клиентов (%, сумма, накопительная); сопротивление менеджеров; необходимость доступа к товарным позициям сделки |

## Скоринг

| Критерий | Оценка (1–5) | Комментарий |
|----------|:---:|-------------|
| A1 — частота боли | 5 | Каждая компания с отделом продаж сталкивается |
| A2 — платёжность/ROI | 5 | Прямая экономия маржи = быстрый ROI |
| A3 — дифференциация | 4 | Конкуренты есть, но фокус на «применение скидки», а не на «контроль» |
| B1 — стандартные объекты | 4 | Сделки + товарные позиции — стандартное API |
| B2 — понятный источник | 5 | Поля скидки в товарных позициях сделки |
| B3 — минимальные права | 3 | Нужен доступ к сделкам + товарам + уведомлениям |
| B4 — без сложной инфраструктуры | 5 | Расчёт при событии изменения сделки |
| C1 — один killer-экран | 5 | Журнал нарушений + настройка порогов |
| C2 — минимум фоновых процессов | 4 | Обработчик событий (не тяжёлый) |
| C3 — производительность | 4 | Расчёт per-deal, не массовый |
| C4 — управляемые edge-cases | 3 | Разные модели скидок — управляемо через настройки |
| **Итого** | **4.3** | Высокий приоритет |

## Красные флаги — НЕТ

- Нет realtime/стриминга
- Ясная ценность (защита маржи)
- Нет тяжёлой инфраструктуры
- Time-to-value < 10 минут (настроить порог → сразу контроль)
