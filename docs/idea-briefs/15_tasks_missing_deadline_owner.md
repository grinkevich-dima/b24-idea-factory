# Идея 15 — Задачи: Задачи без дедлайна/ответственного

## 1) Коротко (одна фраза)
Контроль качества постановки задач: приложение находит задачи без дедлайна, ответственного или других обязательных управленческих полей.

## 2) Для кого (ICP)
- Руководитель проекта
- Тимлид
- Владелец регламента по задачам

## 3) Боль и JTBD
- Задачи создаются без срока или ответственного и теряются в общем списке.
- Руководителю приходится вручную проверять качество постановки.

## 4) Измеримый эффект
- Снижение доли задач без дедлайна и ответственного.
- Повышение предсказуемости выполнения задач.

## 5) УТП (позиционирование)
**“Покажем нарушения постановки задач и мягко вернём их авторам на исправление.”**

## 6) MVP (3–5 фич)
1. **Список задач без дедлайна/ответственного.**
2. **Правила обязательности по проектам и группам.**
3. **Уведомление постановщику о нарушении.**
4. **Отчёт по авторам и проектам с наибольшим числом нарушений.**

## 7) Реализуемость и тех‑чеклист (10 минут)
### Данные и объекты
- Задачи, постановщики, ответственные, дедлайны, группы/проекты.

### Триггеры и обновление
- Проверка новых и изменённых задач, плюс ежедневный контроль активных задач.

### Права и безопасность
- Минимальные права на чтение задач и отправку уведомлений.

### Производительность
- Проверяем только активные задачи и проекты, где включено правило.

## 8) Риски (и как снизить)
- **Не всем задачам нужен дедлайн** → исключения по проектам и типам задач.
- **Сопротивление команды** → мягкий дайджест вместо блокировок.
- **Дублирование штатных фильтров** → упор на правила и отчёт по нарушениям.

## 9) Метрики успеха
- Доля задач без дедлайна.
- Доля задач без ответственного.
- Количество исправленных нарушений после уведомлений.

## 10) Поиск конкурентов (ключевые запросы для Маркета)
- “задачи без дедлайна”
- “задачи без ответственного”
- “качество постановки задач”
- “контроль задач”

## 11) Упаковка для Маркета (черновик)
### Название
Контроль качества постановки задач

### Буллеты ценности
- Находит задачи без дедлайна и ответственного
- Правила по проектам
- Уведомляет постановщиков
- Показывает статистику нарушений
