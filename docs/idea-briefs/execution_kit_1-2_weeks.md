# Execution kit (1–2 недели) — общий шаблон реализации (для всех 5 идей)

Этот файл — “скелет” плана реализации. Для каждой идеи конкретика уже отражена в `*__mvp_release_market.md` и `*__screens_security_faq.md`.

## 1) Объекты и данные (Data model)
- **Конфиг приложения**: правила (SLA/пороги), рабочее время, исключения, получатели уведомлений.
- **Агрегаты**: дневные/недельные сводки для быстрого дашборда.
- **Сущности‑источники**:\n
  - CRM: лид/сделка (+ стадии/воронки)\n
  - Tasks: задача (+ проект/группа, дедлайн, статус)\n
  - IM: только метаданные (если используется), без контента.

## 2) UI точки входа
- Пункт в левом меню: “Дашборд”.
- Страница настроек: “Правила/пороги”.
- Детализация: “Список проблемных элементов” (клик из метрики).

## 3) События / пересчёты
Стратегия по умолчанию (privacy‑first + производительность):
- **Инкрементально по изменениям**, где возможно (смена стадии/статуса/дедлайна).
- **Периодический пересчёт** агрегатов (например 1 раз/час или 1 раз/сутки), только по активным элементам и ограниченному периоду.
- **Кэш**: агрегаты по дням и фильтрам, чтобы дашборд открывался мгновенно.

## 4) Уведомления и эскалации
- Каналы: уведомления Bitrix24 (и/или чат, если используется).
- Правила: “ответственный → руководитель при игноре/повторе”.
- Анти‑шум: лимиты, дайджест 1 раз/день, тихие часы.

## 5) Безопасность и модерация
- Минимизировать права (чтение сущностей + отправка уведомлений).
- Не читать/не хранить содержимое сообщений/писем.
- Хранить только timestamps/агрегаты и настройки.

## 6) План работ (типовой)
### Неделя 1 (MVP ядро)
- Хранилище конфигурации и агрегатов.
- Дашборд + фильтры + детализация списка.
- Базовые настройки порогов/правил.

### Неделя 2 (операционка и полировка)
- Уведомления/эскалации/дайджест.
- Пресеты и пустые состояния.
- Производительность: кэш, ограничения периодов, тест на больших выборках.
- Упаковка: тексты, скриншоты, FAQ.
