# 09 — KPI РОПа: одна панель

## Подготовка (Шаг 0)

**Для руководителей отделов продаж (РОП), которые тратят 30–60 минут в день на сбор данных из разных разделов Bitrix24 — приложение собирает 10–12 ключевых метрик на одном экране, экономя 5+ часов в неделю и давая мгновенную картину состояния отдела.**

## Идея

| Поле | Значение |
|------|----------|
| ICP/роль покупателя | РОП, коммерческий директор (компании с ОП 3–50 менеджеров) |
| Боль | Данные разбросаны: лиды в одном отчёте, сделки в другом, звонки в третьем. Нет единого «пульта» для ежедневного контроля. |
| Измеримый эффект | Экономия 30–60 мин/день на сбор данных; быстрое выявление проблем (менеджер «просел» = видно за 10 сек) |
| MVP-объём | 2 недели: панель с 10–12 метриками (конверсия, количество новых/закрытых сделок, средний чек, просроченные дела, звонки) + фильтры по периоду и менеджеру |
| Основные риски | Конкуренция (много KPI-приложений); необходимость доступа к нескольким сущностям (лиды, сделки, дела, звонки) |

## Скоринг

| Критерий | Оценка (1–5) | Комментарий |
|----------|:---:|-------------|
| A1 — частота боли | 5 | Ежедневная потребность РОПа |
| A2 — платёжность/ROI | 5 | РОП — лицо, принимающее решение о покупке |
| A3 — дифференциация | 2 | Много конкурентов, нужен сильный UX |
| B1 — стандартные объекты | 5 | Лиды, сделки, дела — стандартные |
| B2 — понятный источник | 5 | Все метрики считаются из стандартных полей |
| B3 — минимальные права | 3 | Нужен доступ к CRM (лиды, сделки, дела, звонки) |
| B4 — без сложной инфраструктуры | 4 | Агрегация данных на лету или лёгкий кэш |
| C1 — один killer-экран | 5 | Одна панель — суть продукта |
| C2 — минимум фоновых процессов | 4 | Кэширование агрегатов (опционально) |
| C3 — производительность | 3 | Агрегация по 1000+ сделок — нужна оптимизация |
| C4 — управляемые edge-cases | 4 | Разные воронки/типы сделок — решается фильтрами |
| **Итого** | **4.1** | Средне-высокий приоритет |

## Красные флаги — НЕТ

- Нет realtime-стриминга
- Ясная ценность (экономия времени РОПа)
- Нет тяжёлой инфраструктуры
- Time-to-value: установил → выбрал период → видишь KPI через 1 минуту
