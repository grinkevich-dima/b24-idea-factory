# Идея 23 — Чаты: Дайджест для руководителя

## 1) Коротко (одна фраза)
Ежедневный чат‑дайджест собирает просрочки, зависшие задачи и ключевые риски, чтобы руководитель не открывал несколько отчётов.

## 2) Для кого (ICP)
- Руководитель отдела
- РОП
- Операционный директор

## 3) Боль и JTBD
- Руководитель утром открывает разные разделы Bitrix24, чтобы понять, где горит.
- Важные просрочки теряются среди уведомлений.

## 4) Измеримый эффект
- Сокращение времени ежедневной проверки операционного статуса.
- Более быстрая реакция на просрочки и риски.

## 5) УТП (позиционирование)
**“Один короткий дайджест в чат показывает, что требует внимания сегодня.”**

## 6) MVP (3–5 фич)
1. **Ежедневный дайджест**: просроченные задачи, сделки без движения, ближайшие дедлайны.
2. **Настройка получателей и времени отправки.**
3. **Ссылки на проблемные элементы.**
4. **Ограничение топ‑N, чтобы дайджест был коротким.**

## 7) Реализуемость и тех‑чеклист (10 минут)
### Данные и объекты
- Задачи, сделки, дедлайны, ответственные, агрегаты нарушений.

### Триггеры и обновление
- Отправка по расписанию один раз в день; ручная отправка тестового дайджеста.

### Права и безопасность
- Отправляем только названия/ссылки и агрегаты, без хранения переписки.

### Производительность
- Готовим агрегаты заранее и ограничиваем топ‑N элементов.

## 8) Риски (и как снизить)
- **Дайджест станет слишком длинным** → лимиты и группировка.
- **Дублирование уведомлений** → отдельный канал “только руководителю”.
- **Разные роли хотят разные сигналы** → пресеты дайджеста.

## 9) Метрики успеха
- Открытия ссылок из дайджеста.
- Количество закрытых просрочек после отправки.
- Время руководителя на утреннюю проверку.

## 10) Поиск конкурентов (ключевые запросы для Маркета)
- “дайджест руководителя”
- “ежедневный отчёт”
- “просрочки Bitrix24”
- “чат уведомления”
- “операционный контроль”

## 11) Упаковка для Маркета (черновик)
### Название
Дайджест руководителя в чат

### Буллеты ценности
- Короткий список “где горит”
- Ссылки на проблемные элементы
- Настраиваемые получатели
- Без внешних интеграций
