# Идея 03 — CRM: “Сделки без касаний” — UTP, MVP, релиз, Маркет

## 1) УТП
**Верните в работу “забытые” сделки**: находим сделки без активности N дней, приоритизируем и даём быстрые действия + эскалации.

## 2) MVP (3–5 фич)
1. **Правило “тишины”**: N дней/часов без активности (по воронке/стадии опционально).
2. **Список “без касаний”**: фильтры (ответственный, стадия, сумма, последняя активность) + сортировка по риску.
3. **Режим “Фокус”**: TOP‑10 по сумме/давности (killer‑режим).
4. **Быстрые действия**: открыть сделку / создать задачу / поставить напоминание.
5. **Эскалации**: напоминание → руководителю при игноре.

## 3) Решение ключевой неоднозначности
“Активность” делаем как настройку:\n
- по умолчанию: изменения в сделке (стадия/поля);\n
- опционально: дела/задачи, привязанные к сделке (без чтения содержимого).

## 4) План релиза 1–2 недели
### Неделя 1
- Экран списка + “Фокус” + базовые фильтры.
- Настройки “тишины” + исключения стадий.

### Неделя 2
- Быстрые действия + уведомления/эскалации.
- Метрика “реанимировано” (после активности в 24–48ч).

## 5) Метрики успеха
- Кол-во сделок без касаний > N (в динамике).
- Доля “реанимированных” после напоминаний.
- Среднее время тишины по стадиям/менеджерам.

## 6) Риски
- Ложные срабатывания (пауза/ожидание) → исключения по стадиям/тегам.
- Спам уведомлений → дайджест/лимиты.

## 7) Текст для Маркета (черновик)
### Заголовок
Сделки без касаний: контроль тишины и напоминания

### Подзаголовок
Находит “забытые” сделки и помогает вернуть их в работу за несколько кликов.

### Что вы получаете
- Список сделок без активности за период
- TOP‑10 “Фокус” по важности
- Быстрые действия и эскалации
