# Идея 02 — CRM: SLA по стадиям сделки — UTP, MVP, релиз, Маркет

## 1) УТП (финальная формулировка)
**Ускорьте воронку продаж**: SLA по стадиям + список просрочек + эскалации — руководитель видит, где сделки “застряли” и у кого.

## 2) MVP (3–5 фич)
1. **Настройка SLA по стадиям** (по воронке): порог (часы/дни) на каждую стадию.
2. **Список просроченных сделок**: фильтры (воронка, стадия, ответственный, период, сумма).
3. **Сводка по стадиям**: где накопились просрочки (кол-во, среднее/макс превышение).
4. **Эскалации**: уведомления ответственному и руководителю при превышении порога.
5. **Метки “причина застревания”** (лёгкая версия): 5–10 причин из списка (по желанию).

## 3) Не‑MVP
- Полноценный конструктор отчётов.
- Сложные сценарии “согласований” и вмешательство в канбан.

## 4) План релиза 1–2 недели
### Неделя 1
- Конфиг SLA по воронке/стадиям.
- Экран сводки + экран списка просрочек + детализация.

### Неделя 2
- Уведомления/эскалации + дайджест.
- Метки причин (если успеваем) + экспорт.
- Оптимизация производительности (агрегаты/кэш).

## 5) Метрики успеха
- % активных сделок, превысивших SLA на ключевых стадиях.
- P50/P90 времени на стадии.
- Кол-во устранённых просрочек после алерта.

## 6) Риски
- Разные воронки/настройки у клиентов → конфиг по выбранной воронке, пресеты “выключено”.
- Шум уведомлений → лимиты, дайджест, тихие часы.

## 7) Текст для Маркета (черновик)
### Заголовок
SLA по стадиям сделок: просрочки и ускорение воронки

### Подзаголовок
Показывает “застревания” по стадиям и автоматически эскалирует нарушения SLA.

### Что внутри
- SLA‑пороги по стадиям (по воронке)
- Список просроченных сделок с фильтрами
- Сводка “где горит” по стадиям
- Уведомления и эскалации

### Быстрый старт
Установите → выберите воронку → задайте SLA по стадиям → контролируйте просрочки.
