# 06 — Скидки под контролем: анализ конкурентов

## Методология поиска

Ключевые запросы в Bitrix24 Marketplace: «скидки», «управление скидками», «маржа», «контроль маржинальности», «скидка сделка», «discount», «наценка», «ценообразование», «контроль цен», «margin control».

## Конкуренты

### 1. Управление скидками (ООО «Бизнес Интеграция»)

| Параметр | Значение |
|----------|----------|
| Обещание | Гибкая настройка скидок через роботов бизнес-процессов |
| Механика | 2 робота: скидка в % и скидка в сумме; применяются к товарным позициям в сделках/лидах/счетах |
| Installs/оценка | ~500+ установок |
| Запрашиваемые права | Полный доступ к CRM (лиды, сделки, счета, товары) |
| Пробелы | **Применяет** скидки, но **не контролирует** превышение. Нет журнала нарушений, нет порогов, нет алертов руководителю |

### 2. Умные финансы в Сделках

| Параметр | Значение |
|----------|----------|
| Обещание | Автоматический расчёт расходов, доходов и маржи в сделке |
| Механика | Поля доходов/расходов + формула маржи + сценарии (показать маржу, вычислить задолженность) |
| Installs/оценка | 1000+ установок |
| Запрашиваемые права | Полный доступ к сделкам |
| Пробелы | Считает маржу **постфактум**, но не **предотвращает** выдачу чрезмерной скидки. Нет правил/порогов, нет автоматических алертов |

### 3. Скидка на все товары (Активити)

| Параметр | Значение |
|----------|----------|
| Обещание | Массовое применение скидки на все товары в лиде/сделке |
| Механика | Активити для бизнес-процессов; скидка в % распределяется на каждый товар |
| Installs/оценка | ~300+ установок |
| Запрашиваемые права | Доступ к товарным позициям |
| Пробелы | Только **применение** скидки, без ограничений. Нет лимитов, нет логирования, нет ролевой модели |

### 4. Индивидуальные решения (it-solution.ru и др.)

| Параметр | Значение |
|----------|----------|
| Обещание | Автоматический контроль маржинальности и блокировка чрезмерных скидок |
| Механика | Заказная разработка: правила порогов + блокировка + отчёт |
| Installs/оценка | Не публичное приложение (кастом) |
| Запрашиваемые права | Индивидуально |
| Пробелы | Недоступно как публичное приложение. Дорого. Не масштабируется на рынок. |

### 5. Расходы по сделкам / Управление расходами и доходами

| Параметр | Значение |
|----------|----------|
| Обещание | Учёт затрат и расчёт прибыли по сделке |
| Механика | Поля расходов + автоматический расчёт |
| Installs/оценка | 500+ установок |
| Запрашиваемые права | Доступ к сделкам |
| Пробелы | Фокус на расходах, а не на скидках. Нет превентивного контроля. Нет уведомлений о нарушении правил |

## Топ слабых мест рынка

1. **Нет превентивного контроля** — все решения работают постфактум (применяют скидку или считают маржу), никто не блокирует/алертит ДО того, как скидка станет проблемой.
2. **Нет журнала отклонений** — руководитель не видит историю «кто, когда, на сколько превысил порог».
3. **Нет ролевой модели** — нельзя задать разные пороги для разных менеджеров/должностей.
4. **Нет эскалации** — нет автоматического уведомления руководителю при нарушении.
5. **Избыточные права** — конкуренты запрашивают полный доступ к CRM, хотя для контроля достаточно чтения + уведомлений.

## Что мы делаем лучше

- **Превентивный контроль**: правила срабатывают в момент сохранения сделки, ДО закрытия.
- **Журнал нарушений**: полная история отклонений с фильтрами.
- **Гибкие пороги**: по воронке, стадии, роли менеджера, типу клиента.
- **Мгновенный алерт руководителю** в чат при превышении порога.
- **Минимальные права**: чтение товарных позиций + уведомления (не полный доступ к CRM).
