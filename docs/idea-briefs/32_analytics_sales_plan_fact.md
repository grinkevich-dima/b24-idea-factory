# Идея 32 — Аналитика: План продаж против факта по менеджерам

## 1) Коротко (одна фраза)
Панель сравнивает план продаж с фактом по менеджерам, воронкам и периодам без внедрения отдельной BI‑системы.

## 2) Для кого (ICP)
- РОП
- Коммерческий директор
- Собственник бизнеса

## 3) Боль и JTBD
- План продаж ведётся отдельно, а факт живёт в CRM.
- Отклонения по менеджерам становятся видны слишком поздно.

## 4) Измеримый эффект
- Более ранний контроль недовыполнения плана.
- Сокращение ручного сведения план/факт отчётов.

## 5) УТП (позиционирование)
**“План и факт продаж по менеджерам в одной панели Bitrix24.”**

## 6) MVP (3–5 фич)
1. **Ручной ввод планов по менеджерам и периодам.**
2. **Факт по выигранным сделкам из CRM.**
3. **Процент выполнения и прогноз до конца периода.**
4. **Детализация по менеджерам и воронкам.**

## 7) Реализуемость и тех‑чеклист (10 минут)
### Данные и объекты
- Сделки, суммы, стадии выигрыша, ответственные, периоды планов.

### Триггеры и обновление
- Пересчёт факта по расписанию и при открытии панели.

### Права и безопасность
- Храним планы и агрегаты; не читаем содержимое коммуникаций.

### Производительность
- Агрегация факта по периодам и менеджерам, ограничение выбранным периодом.

## 8) Риски (и как снизить)
- **Сложные схемы планирования** → MVP только план по менеджеру/месяцу.
- **Спор по дате факта** → настройка: дата закрытия или оплаты, если поле есть.
- **Разные валюты** → в MVP использовать валюту портала/сделки без сложной конвертации.

## 9) Метрики успеха
- Процент выполнения плана.
- Количество менеджеров в зоне риска.
- Время подготовки план/факт отчёта.

## 10) Поиск конкурентов (ключевые запросы для Маркета)
- “план факт продаж”
- “план менеджера”
- “KPI продаж”
- “выполнение плана”
- “CRM аналитика”

## 11) Упаковка для Маркета (черновик)
### Название
План продаж против факта

### Буллеты ценности
- Планы и факт в одной панели
- Процент выполнения по менеджерам
- Прогноз до конца периода
- Без отдельного BI‑проекта
