# Идея 01 — CRM: SLA реакции на лид — UTP, MVP, релиз, Маркет

## 1) УТП (финальная формулировка)
**Сократите время до первого касания по лидам**: дашборд просрочек + правила SLA + эскалации — видно, кто тормозит и где “утекают” лиды.

## 2) MVP (строго 3–5 фич)
1. **SLA‑дашборд**: “в зоне риска / просрочено / ок” + фильтры (источник, ответственный, период).
2. **Определение “первого касания” (переключатель)**:\n
   - вариант A: “первая смена статуса/стадии лида”;\n
   - вариант B: “первое дело/активность” (если доступно без расширенных прав/сложности).
3. **Правила SLA**: порог в мин/час + рабочие часы (упрощённо) + исключения (выходные/ночь).
4. **Эскалации**: уведомление ответственному → руководителю при превышении SLA; режим дайджеста (1 раз/день).
5. **Детализация**: клик из метрики → список лидов + переход в карточку.

## 3) Не‑MVP (сразу выкинуть, чтобы уложиться)
- ML/анализ текста/голоса, чтение содержания сообщений.
- Сложные “мульти‑правила” по сегментам/AB‑тесты.
- Полная BI‑аналитика и десятки вкладок.

## 4) Реализуемость (решения по спорным местам)
- **Счёт “до первого касания”**: дефолт — “первая смена статуса”, т.к. это проще объяснить, не требует чтения коммуникаций.\n
  Второй режим (“первое дело/активность”) — как опция.
- **Права**: делаем режим “privacy‑first”: без чтения содержимого коммуникаций, храним только метрики и timestamps.

## 5) План релиза на 1–2 недели (объём работ)
### Неделя 1
- Модель данных/хранилище метрик (timestamp создания, timestamp первого касания, SLA‑порог, статус).
- Экран дашборда + фильтры + детализация списка.
- Настройки SLA (порог + рабочие часы упрощённо).

### Неделя 2
- Уведомления и эскалации + дайджест.
- Полировка UX: пресеты SLA, пустые состояния, быстрые ссылки.
- Мини‑набор тест‑сценариев на больших выборках и права.

## 6) Метрики успеха (что мерить)
- % лидов с реакцией ≤ SLA.
- P50/P90 времени до первого касания.
- Сколько просрочек “закрыто” после алертов (активность в 24ч).

## 7) Риски
- Разное понимание “касания” → опция выбора метода + дефолт.
- Шум от уведомлений → лимиты, дайджест, тихие часы.

## 8) Текст для Маркета (черновик)
### Заголовок
SLA по лидам: контроль скорости реакции

### Подзаголовок
Покажет просрочки по лидам и автоматически эскалирует нарушения SLA.

### Кому подходит
- РОП и руководителям групп продаж
- Лид‑менеджерам/колл‑центрам

### Что вы получаете
- Дашборд “в зоне риска / просрочено / ок”
- Настраиваемые пороги SLA (мин/часы) и рабочее время
- Эскалации ответственным и руководителям
- Детализация до конкретных лидов

### Как это работает (3 шага)
1) Установите приложение.\n
2) Выберите правило “первого касания” и порог SLA.\n
3) Контролируйте просрочки и реагируйте на алерты.

### FAQ (коротко)
- **Читает ли приложение тексты сообщений?** Нет, мы считаем только метрики времени.\n
- **Сколько времени до пользы?** Первый отчёт — сразу после установки, максимум после выбора правил.
