# Идея 12 — Задачи: Перегруз исполнителей (WIP)

## 1) Коротко (одна фраза)
Отчёт показывает сотрудников с избыточным количеством активных задач и помогает перераспределять нагрузку до срыва сроков.

## 2) Для кого (ICP)
- Руководитель отдела
- Руководитель проекта
- Тимлид операционной команды

## 3) Боль и JTBD
- У одних сотрудников копится слишком много задач, а другие недозагружены.
- Перегруз становится виден только после просрочек и конфликтов по приоритетам.

## 4) Измеримый эффект
- Снижение количества просроченных задач у перегруженных исполнителей.
- Более ровное распределение активных задач по команде.

## 5) УТП (позиционирование)
**“Покажем перегруз исполнителей по активным задачам без ручного свода в таблицах.”**

## 6) MVP (3–5 фич)
1. **Панель WIP**: активные задачи по исполнителям и группам.
2. **Пороги перегруза по количеству задач и ближайшим дедлайнам.**
3. **Список задач, которые формируют перегруз.**
4. **Еженедельный дайджест руководителю.**

## 7) Реализуемость и тех‑чеклист (10 минут)
### Данные и объекты
- Задачи, ответственные, статусы, дедлайны, группы/проекты.

### Триггеры и обновление
- Периодический пересчёт агрегатов и ручное обновление из панели.

### Права и безопасность
- Только чтение задач и отправка уведомлений; содержимое комментариев не читаем и не сохраняем.

### Производительность
- Ограничение активными задачами и фильтрами по группе, ответственному и периоду дедлайна.

## 8) Риски (и как снизить)
- **Спор о норме нагрузки** → настраиваемые пороги по командам.
- **Разная сложность задач** → MVP считает количество и дедлайны, без оценки трудоёмкости.
- **Шум уведомлений** → дайджест вместо частых алертов.

## 9) Метрики успеха
- Количество сотрудников выше WIP‑порога.
- Доля задач в просрочке у перегруженных исполнителей.
- Количество перераспределённых задач после просмотра панели.

## 10) Поиск конкурентов (ключевые запросы для Маркета)
- “перегруз задач”
- “WIP”
- “нагрузка исполнителей”
- “балансировка задач”
- “контроль загрузки”

## 11) Упаковка для Маркета (черновик)
### Название
Перегруз исполнителей: WIP‑контроль задач

### Буллеты ценности
- Видите перегруз до просрочек
- Пороги нагрузки по командам
- Детализация до конкретных задач
- Дайджест руководителю
