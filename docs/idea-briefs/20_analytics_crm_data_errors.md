# Идея 20 — Аналитика: Качество CRM: ошибки данных

## 1) Коротко (одна фраза)
Отчёт находит типовые ошибки в CRM‑данных: пустые суммы, странные даты, незаполненные обязательные поля и битые значения.

## 2) Для кого (ICP)
- Владелец CRM
- РОП
- CRM‑администратор

## 3) Боль и JTBD
- Отчёты и прогнозы ломаются из‑за мусорных данных.
- Ошибки находятся вручную или уже после управленческого отчёта.

## 4) Измеримый эффект
- Снижение доли карточек с ошибками данных.
- Повышение доверия к CRM‑аналитике.

## 5) УТП (позиционирование)
**“Покажем ошибки CRM‑данных и дадим ссылки на исправление без сложного аудита.”**

## 6) MVP (3–5 фич)
1. **Профили проверок**: сумма, даты, источник, ответственный, обязательные поля.
2. **Список найденных ошибок со ссылками на карточки.**
3. **Скоринг качества по менеджерам и воронкам.**
4. **Еженедельный отчёт владельцу CRM.**

## 7) Реализуемость и тех‑чеклист (10 минут)
### Данные и объекты
- Лиды, сделки, компании/контакты, пользовательские поля по настройке.

### Триггеры и обновление
- Периодический скан активных карточек и ручной запуск проверки.

### Права и безопасность
- Читаем только выбранные поля; храним агрегаты и идентификаторы нарушений.

### Производительность
- Проверяем ограниченный период и активные воронки, используем пагинацию.

## 8) Риски (и как снизить)
- **У клиентов разные поля** → настраиваемые профили проверок.
- **Много ложных нарушений** → режим “только отчёт” и исключения.
- **Большой объём данных** → проверка батчами.

## 9) Метрики успеха
- Доля карточек с ошибками.
- Количество исправленных нарушений.
- Скоринг качества по воронкам.

## 10) Поиск конкурентов (ключевые запросы для Маркета)
- “качество CRM”
- “ошибки данных”
- “аудит CRM”
- “заполненность CRM”
- “CRM отчёт”

## 11) Упаковка для Маркета (черновик)
### Название
Качество CRM: ошибки данных

### Буллеты ценности
- Находит пустые и некорректные поля
- Скоринг качества CRM
- Ссылки на исправление
- Еженедельный отчёт владельцу CRM
