# Идея 04 — CRM: Data Quality Score — UTP, MVP, релиз, Маркет

## 1) УТП
**Прозрачные требования к полям + скоринг качества данных**: видно, что не заполнено, у кого, и как это влияет на процесс.

## 2) MVP (3–5 фич)
1. **Профиль требований** для 1 сущности (старт: сделки или лиды): обязательные поля по стадиям.
2. **Скоринг качества**: % заполненности + список критичных пропусков.
3. **Отчёт нарушений**: по менеджерам/стадиям + фильтры.
4. **Напоминания**: уведомления ответственным (мягко), эскалация при систематике.
5. (Опционально) **Предупреждение при переводе стадии** (без жёсткой блокировки).

## 3) План релиза 1–2 недели
### Неделя 1
- Конструктор профиля требований (простая форма).
- Дашборд качества + список нарушений.

### Неделя 2
- Напоминания/эскалации.
- Расширение профиля на вторую сущность (если успеваем) или улучшение фильтров/UX.

## 4) Метрики успеха
- % карточек без критичных пропусков на ключевых стадиях.
- Время “до исправления” после напоминания.

## 5) Риски
- Разнообразие кастом‑полей → начать с пресетов и самых массовых типов полей.
- Сопротивление пользователей → только “мягкий режим” в MVP.

## 6) Текст для Маркета (черновик)
### Заголовок
Data Quality для CRM: контроль обязательных полей

### Подзаголовок
Скоринг качества данных и список нарушений по менеджерам и стадиям.

### Что внутри
- Требования к полям по стадиям
- Скоринг и критичные пропуски
- Напоминания и отчёты
