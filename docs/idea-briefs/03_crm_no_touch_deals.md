# Идея 03 — CRM: “Сделки без касаний” (нет активности N дней)

## 1) Коротко (одна фраза)
Панель, которая показывает сделки **без активности/касаний** за выбранный период и автоматически напоминает ответственным.

## 2) Для кого (ICP)
- РОП / руководитель группы продаж
- Владелец CRM (дисциплина процесса)

## 3) Боль и JTBD
- Сделки стоят, потому что менеджер “не дожимает”, а руководитель не видит это вовремя.
- Нет простого правила “если N дней тишина — бей тревогу”.

## 4) Измеримый эффект
- Снижение доли “мертвых” сделок.
- Ускорение переходов по стадиям (вторично).

## 5) УТП (позиционирование)
**“Мгновенно находим сделки без касаний и возвращаем их в работу без ручной проверки CRM.”**

## 6) MVP (3–5 фич)
1. **Правила “тишины”**: N часов/дней без активности (по воронке/стадии опционально).
2. **Список сделок без касаний**: фильтры по ответственным/стадиям/сумме/сроку.
3. **Напоминания**: уведомления ответственному, эскалация руководителю при игноре.
4. **KPI**: количество “реанимированных” сделок после напоминаний.
5. (Опционально) **Исключения**: стадии “на паузе”, сделки “ожидание клиента” и т.п.

## 7) Реализуемость и тех‑чеклист (10 минут)
### Данные и объекты
- Сущность: **сделка**.
- Нужно определение “активности” (сделать настройкой).

### “Активность” — простые варианты для MVP
- Изменение стадии/суммы/ключевых полей сделки.
- Создание/закрытие задач, привязанных к сделке (если доступно просто).
- Добавление комментария/дела (если не требует широких прав на содержимое).

### Триггеры и обновление
- Можно реализовать **без событий**: раз в X часов считаем “последняя активность” по активным сделкам (по фильтрам), формируем список/алерты.
- Лучше: дополнять пересчёт событиями изменения сделки/задач.

### Права и безопасность
- Работать в рамках прав пользователя на сделки.
- Не хранить тексты, хранить только timestamp последней активности и тип.

### Производительность
- Ограничить выборку “активные сделки” + период.
- Кэшировать “последнюю активность” инкрементально.

### UI‑объём
- Экран 1: список “без касаний”.
- Экран 2: правила.

## 8) Риски (и как снизить)
- **Спорность “что считать касанием”** → настройка + 1–2 дефолта.
- **Ложные срабатывания** (сделка на паузе) → исключения по стадиям/тегам.
- **Спам уведомлений** → дайджест, лимиты, тихие часы.

## 9) Метрики успеха
- Кол-во сделок без касаний > N.
- Среднее время “тишины” по стадиям.
- Доля сделок, по которым после алерта была активность в течение 24–48 часов.

## 10) Поиск конкурентов (ключевые запросы для Маркета)
- “сделки без активности”
- “контроль касаний”
- “просроченные сделки”
- “план касаний”
- “напоминания по сделкам”
- “KPI менеджера продажи”

## 11) Упаковка для Маркета (черновик)
### Название
Сделки без касаний: контроль тишины и напоминания

### Буллеты ценности
- Находите “мертвые” сделки за 1 экран
- Настраиваемые правила тишины по сделкам
- Напоминания и эскалации без ручного контроля CRM
- Минимум настроек, быстрый результат
