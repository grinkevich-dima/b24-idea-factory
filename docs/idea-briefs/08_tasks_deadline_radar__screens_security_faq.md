# 08 — Просрочки по дедлайнам: радар — скриншоты + безопасность + FAQ

## План скриншотов (7)

1. **Главный экран — Радар**: 3 зоны в виде карточек/секторов: 🔴 «Просрочено: 8» / 🟡 «Дедлайн завтра: 3» / 🟠 «Эта неделя: 12». Визуально — от красного к зелёному. Общий индикатор здоровья.
2. **Детализация красной зоны**: список просроченных задач — название, проект, ответственный, на сколько дней просрочено. Сортировка по критичности.
3. **Детализация жёлтой зоны**: задачи с дедлайном завтра — аналогичный список, но с акцентом «успей сегодня».
4. **Фильтры**: применён фильтр «Проект: Разработка v2.0» — показывает только задачи этого проекта в 3 зонах.
5. **Алерт в чате**: сообщение: «🚨 Критический уровень просрочек: 15 задач в красной зоне (порог: 10). Требуется внимание. [Открыть радар]».
6. **Настройки**: порог алерта (число задач), получатели, исключённые проекты, частота обновления.
7. **Пустое состояние (всё ок)**: «✅ Все дедлайны под контролем. 0 просрочек, 2 задачи завтра.» — зелёный индикатор.

## Стратегия минимальных прав

| Право | Обоснование | Уровень |
|-------|-------------|---------|
| task.item.read | Чтение задач (дедлайн, статус, проект, ответственный) | Чтение |
| im.message.add | Отправка алертов | Запись (только сообщения) |
| user.basic | Имена ответственных | Чтение |
| sonet_group.read | Список проектов (для фильтра) | Чтение |

**Не запрашиваем:**
- Запись/изменение задач
- Доступ к CRM
- Полный доступ к чатам
- Доступ к файлам
- Доступ к описаниям/комментариям задач

## Что приложение НЕ читает / НЕ хранит

- ❌ Описание и комментарии задач
- ❌ Вложенные файлы
- ❌ Данные CRM, переписки, звонки
- ❌ Персональные данные клиентов
- ❌ Содержимое чатов

## Политика хранения

- Храним: кэш агрегатов (количество задач по зонам, ID задач), обновляется каждые 15 минут
- Срок кэша: 1 час (затем переформируется)
- Не храним: историю просрочек (показываем только текущее состояние)
- При удалении: все данные удаляются мгновенно

## Производительность

- Обновление данных: каждые 15 минут (настраивается: 5/15/30/60 мин)
- Запрос: `task.item.list` с фильтрами STATUS=активные + DEADLINE <= now+7 дней
- Кэширование: на уровне приложения, не нагружает API между обновлениями
- Лимит: топ-50 задач на зону (для больших порталов)
- Первая загрузка: < 3 секунды (до 1000 задач с дедлайном на неделю)

## FAQ (текст для Маркета)

**Вопрос: Чем отличается от встроенного фильтра «просроченные»?**
Ответ: Радар показывает ВСЕ проекты на одном экране, добавляет «предупреждающие» зоны (завтра, неделя) и агрегирует — сразу видно масштаб проблемы, а не список из 200 задач.

**Вопрос: Как работает на портале с 10 000+ задач?**
Ответ: Фильтруем только активные задачи с дедлайном в ближайшие 7 дней + просроченные. Кэширование на 15 мин. Топ-50 на зону. Нагрузки на портал нет.

**Вопрос: Задачи без дедлайна тоже показываются?**
Ответ: Отдельным счётчиком: «Без дедлайна: 45 задач». Не попадают в основные зоны, но напоминают о необходимости поставить срок.

**Вопрос: Можно ли исключить определённые проекты?**
Ответ: Да. В настройках — список исключений. Например, «Бэклог» или «Идеи» — проекты без строгих дедлайнов.

**Вопрос: Приложение может продлить дедлайн?**
Ответ: Нет. Только информирует. Не изменяет задачи — для безопасности и минимальных прав.

**Вопрос: Какие данные хранятся?**
Ответ: Только кэш количества задач по зонам (обновляется каждые 15 мин). Не храним содержимое задач, файлы, комментарии или персональные данные.
