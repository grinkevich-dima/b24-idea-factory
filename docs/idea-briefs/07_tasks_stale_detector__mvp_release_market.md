# 07 — Зависшие задачи без изменений: MVP + релиз + текст Маркета

## УТП

**Единственное приложение, которое находит «тихо умирающие» задачи — без изменений N дней — и доставляет список руководителю в чат, пока проблема не стала критической.**

## MVP (3–5 фич)

1. **Детектор тишины** — поиск задач без изменений (комментарий, смена статуса, вложение) за последние N дней (настраивается: 3/5/7/14).
2. **Список зависших** — экран со списком задач, отсортированных по «дням молчания» (desc), с фильтрами по проекту/ответственному.
3. **Дайджест в чат** — автоматическое сообщение руководителю (ежедневно или еженедельно) с топ-10 зависших задач.
4. **Исключения** — возможность пометить задачу как «ожидаемо неактивна» (чтобы не попадала в дайджест).
5. **Настройки** — порог дней, расписание дайджеста, получатели, исключённые проекты.

## Не-MVP (отложено)

- Автоматическое назначение ответственного при зависании
- Интеграция с Канбан (перенос на стадию «Внимание»)
- Аналитика трендов (% зависших задач по неделям)
- Групповые действия (массовое закрытие/переназначение)
- Уведомление исполнителю (а не только руководителю)

## План релиза

### Неделя 1
- День 1–2: API-запросы к задачам + логика определения «тишины» (CHANGED_DATE < now - N дней)
- День 3–4: UI списка зависших задач (таблица + фильтры + сортировка)
- День 5: Настройки (порог дней, исключения)

### Неделя 2 (опционально, можно уложиться в 1 неделю)
- День 1–2: Механизм дайджеста (cron + форматирование сообщения + отправка в чат)
- День 3: Тестирование на тестовом портале
- День 4–5: Публикация + тексты Маркета

## Риски и смягчения

| Риск | Смягчение |
|------|-----------|
| Шум при первом запуске (старые задачи) | Фильтр: показывать только задачи, созданные за последние 90 дней (настраивается) |
| Определение «активности» неоднозначно | MVP: CHANGED_DATE — включает любое изменение. Достаточно для 90% случаев |
| Производительность на больших порталах | Пагинация + фильтр по STATUS (только активные) + лимит 500 задач |

## Черновик текста для Маркета

### Заголовок
**Зависшие задачи — детектор «болота»**

### Подзаголовок
Находит задачи без активности N дней и отправляет дайджест руководителю. Никаких забытых задач.

### Буллеты ценности
- ✅ Находит задачи, которые «молчат» 3/5/7/14 дней — пока стандартные фильтры их не видят
- ✅ Ежедневный/еженедельный дайджест в чат — не нужно заходить в приложение
- ✅ Работает для ВСЕХ задач — не только проектных, не только CRM
- ✅ Установка → результат за 1 минуту (без настройки роботов/триггеров)
- ✅ Минимальные права — только чтение задач + отправка уведомлений

### Quick Start
1. Установите приложение
2. Задайте порог «дней молчания» (по умолчанию: 7 дней)
3. Выберите получателя дайджеста
4. Через 1 минуту увидите список зависших задач

### FAQ
- **Чем отличается от фильтра «просроченные»?** — Просроченная = пропущен дедлайн. Зависшая = нет никакой активности N дней (даже если дедлайн не наступил).
- **Какие изменения считаются «активностью»?** — Любое: комментарий, смена статуса, вложение, изменение полей.
- **Что если при первом запуске 500 зависших задач?** — Фильтруем по дате создания (последние 90 дней) + показываем топ по «дням молчания».
