# 07 — Зависшие задачи без изменений

## Подготовка (Шаг 0)

**Для руководителей проектов и тимлидов, у которых задачи «зависают» без активности в Bitrix24 — приложение сокращает количество застоявшихся задач на 50%+ за счёт автоматического выявления и дайджеста руководителю.**

## Идея

| Поле | Значение |
|------|----------|
| ICP/роль покупателя | Руководитель проекта, тимлид, РОП (компании 10–500 чел.) |
| Боль | Задачи ставятся, но «забываются» — нет изменений 5–14 дней. Руководитель не видит это в стандартных фильтрах, пока не станет поздно. |
| Измеримый эффект | Сокращение «болота» (задач без активности > N дней) на 50%+; ускорение среднего цикла задачи |
| MVP-объём | 1 неделя: список зависших задач с фильтрами + еженедельный дайджест руководителю в чат |
| Основные риски | Определение «активности» (комментарий? смена статуса? вложение?); шум при первом запуске (много старых задач) |

## Скоринг

| Критерий | Оценка (1–5) | Комментарий |
|----------|:---:|-------------|
| A1 — частота боли | 5 | Универсальная проблема любой команды |
| A2 — платёжность/ROI | 3 | Косвенный ROI (экономия времени руководителя) |
| A3 — дифференциация | 4 | Нет прямого аналога «stale detector» на Маркете |
| B1 — стандартные объекты | 5 | Задачи — стандартный модуль, API полное |
| B2 — понятный источник | 5 | Дата последнего изменения задачи (CHANGED_DATE) |
| B3 — минимальные права | 4 | Чтение задач + отправка уведомлений |
| B4 — без сложной инфраструктуры | 5 | Периодический запрос к API + фильтрация |
| C1 — один killer-экран | 5 | Список зависших задач с сортировкой по «дням молчания» |
| C2 — минимум фоновых процессов | 5 | Лёгкий cron (раз в день) |
| C3 — производительность | 4 | Пагинация, фильтр по дате — не проблема |
| C4 — управляемые edge-cases | 4 | Исключить завершённые/отложенные — просто |
| **Итого** | **4.5** | Высокий приоритет |

## Красные флаги — НЕТ

- Нет realtime
- Ясная ценность
- Лёгкая инфраструктура
- Time-to-value: установил → видишь список через 1 минуту
