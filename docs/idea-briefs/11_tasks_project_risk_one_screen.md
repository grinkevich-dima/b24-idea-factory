# Идея 11 — Задачи: Риск проекта за 1 экран

## 1) Коротко (одна фраза)
Панель, которая собирает ключевые сигналы из задач проекта и показывает риск срыва сроков до того, как проблема станет очевидной.

## 2) Для кого (ICP)
- Руководитель проекта
- Тимлид команды внедрения/разработки
- Операционный директор, который смотрит портфель проектов

## 3) Боль и JTBD
- Проект выглядит нормально, пока внезапно не появляется критическая просрочка.
- Руководитель вручную открывает группы, задачи и комментарии, чтобы понять, где горит.

## 4) Измеримый эффект
- Снижение доли проектов с внезапными просрочками.
- Сокращение времени еженедельной проверки статуса проектов.

## 5) УТП (позиционирование)
**“Один экран показывает проектные риски по задачам за 10 минут после установки.”**

## 6) MVP (3–5 фич)
1. **Риск‑скоринг проекта по 4 сигналам**: просрочки, задачи без движения, перегруз исполнителей, дедлайны на ближайшие дни.
2. **Список проектов с сортировкой по риску.**
3. **Детализация**: какие задачи формируют риск.
4. **Порог риска и ежедневный дайджест руководителю.**

## 7) Реализуемость и тех‑чеклист (10 минут)
### Данные и объекты
- Задачи, проекты/группы, ответственные, дедлайны, статусы, даты изменения.

### Триггеры и обновление
- Периодический пересчёт 1–2 раза в день и пересчёт по запросу на главном экране.

### Права и безопасность
- Читаем только метаданные задач в рамках прав пользователя; не храним тексты комментариев.

### Производительность
- По умолчанию считаем активные задачи за последние 90 дней и кэшируем агрегаты по проектам.

## 8) Риски (и как снизить)
- **Спорный вес факторов** → дать простые настройки весов и дефолтный пресет.
- **Слишком много красных проектов** → показывать топ‑10 и объяснение причины риска.
- **Большие порталы** → ограничение периода и пагинация детализации.

## 9) Метрики успеха
- Количество проектов в красной зоне.
- Доля рисков, закрытых после дайджеста.
- Время просмотра портфеля проектов руководителем.

## 10) Поиск конкурентов (ключевые запросы для Маркета)
- “риск проекта”
- “здоровье проекта”
- “просроченные задачи”
- “проектный дашборд”
- “контроль задач”

## 11) Упаковка для Маркета (черновик)
### Название
Риск проекта: один экран по задачам

### Буллеты ценности
- Рейтинг проектов по риску
- Понятные причины риска по каждой группе
- Дайджест руководителю без ручной проверки
- Быстрый старт без сложной настройки
