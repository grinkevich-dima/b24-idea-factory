# Идея 14 — Задачи: Просрочки по дедлайнам: радар

## 1) Коротко (одна фраза)
Радар показывает задачи с просроченными и близкими дедлайнами, чтобы команда видела угрозы срокам заранее.

## 2) Для кого (ICP)
- Руководитель проекта
- Тимлид
- Операционный руководитель

## 3) Боль и JTBD
- Просрочки обнаруживаются после факта.
- Руководитель вручную просматривает разные проекты, чтобы понять ближайшие риски.

## 4) Измеримый эффект
- Снижение доли задач, ушедших в просрочку.
- Рост количества задач, закрытых до дедлайна.

## 5) УТП (позиционирование)
**“Один радар показывает просрочки и задачи в зоне риска по всем проектам.”**

## 6) MVP (3–5 фич)
1. **Дашборд**: просрочено, сегодня, 1–3 дня до дедлайна.
2. **Детальный список задач с ответственными.**
3. **Пороги зоны риска по проектам.**
4. **Уведомления ответственным и дайджест руководителю.**

## 7) Реализуемость и тех‑чеклист (10 минут)
### Данные и объекты
- Задачи, дедлайны, статусы, ответственные, группы/проекты.

### Триггеры и обновление
- Пересчёт по расписанию и при открытии панели.

### Права и безопасность
- Читаем только поля задач, доступные пользователю; не сохраняем содержимое задач сверх агрегатов.

### Производительность
- Фильтры по активным задачам и ближайшему периоду дедлайнов.

## 8) Риски (и как снизить)
- **Слишком много уведомлений** → группировать в дайджест.
- **Дедлайны ставят не всегда** → отдельный фильтр “без дедлайна”.
- **Большой объём задач** → кэшировать сводки по дням.

## 9) Метрики успеха
- Количество просроченных задач.
- Количество задач в зоне риска.
- Доля задач закрытых до дедлайна.

## 10) Поиск конкурентов (ключевые запросы для Маркета)
- “просрочки задач”
- “дедлайны задач”
- “радар дедлайнов”
- “контроль сроков”
- “задачи Bitrix24”

## 11) Упаковка для Маркета (черновик)
### Название
Радар дедлайнов по задачам

### Буллеты ценности
- Просрочки и ближайшие дедлайны в одном экране
- Фильтры по проектам и ответственным
- Дайджест руководителю
- Быстрые ссылки на задачи
