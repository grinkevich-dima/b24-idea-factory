# 08 — Просрочки по дедлайнам: радар

## Подготовка (Шаг 0)

**Для руководителей проектов и менеджеров, которые не могут быстро оценить масштаб просрочек в Bitrix24 — приложение даёт единый экран «радар» с визуализацией всех приближающихся и уже нарушенных дедлайнов, сокращая реакцию на просрочку с дней до минут.**

## Идея

| Поле | Значение |
|------|----------|
| ICP/роль покупателя | Руководитель проекта, PMO, тимлид (компании 15–1000 чел.) |
| Боль | Стандартные фильтры показывают просроченные задачи, но не дают «радарной» картины: что горит сейчас, что загорится завтра, где тренд ухудшается. |
| Измеримый эффект | Сокращение среднего времени реакции на просрочку; снижение % просроченных задач на 30–40% |
| MVP-объём | 1–2 недели: дашборд (зоны: «горит» / «завтра» / «эта неделя»), алерты, детализация до задачи |
| Основные риски | Производительность на порталах с 10 000+ задач; информационный шум при плохой дисциплине дедлайнов |

## Скоринг

| Критерий | Оценка (1–5) | Комментарий |
|----------|:---:|-------------|
| A1 — частота боли | 5 | Просрочки — повседневная проблема |
| A2 — платёжность/ROI | 4 | Экономия времени руководителя + снижение срывов |
| A3 — дифференциация | 3 | Встроенный функционал частично покрывает, но без «радарного» UX |
| B1 — стандартные объекты | 5 | Задачи — стандартное API |
| B2 — понятный источник | 5 | Поля DEADLINE, STATUS — однозначные |
| B3 — минимальные права | 4 | Чтение задач + уведомления |
| B4 — без сложной инфраструктуры | 5 | Запросы к API + визуализация |
| C1 — один killer-экран | 5 | Радар с зонами — главный экран |
| C2 — минимум фоновых процессов | 4 | Периодическое обновление данных |
| C3 — производительность | 3 | Нужна пагинация и кэширование для больших порталов |
| C4 — управляемые edge-cases | 4 | Задачи без дедлайна — исключить или подсветить отдельно |
| **Итого** | **4.3** | Высокий приоритет |

## Красные флаги — НЕТ

- Нет realtime-стриминга (периодические запросы)
- Ясная ценность для руководителя
- Простая инфраструктура
- Time-to-value: установил → видишь радар через 2 минуты
