# 10 — Дайджест для руководителя: скриншоты + безопасность + FAQ

## План скриншотов (7)

1. **Дайджест в чате (главный скриншот)**: форматированное сообщение в чате Bitrix24:
   ```
   📋 Утренний дайджест (28 мая, 09:00)
   
   🔴 Просроченные задачи (5):
   1. Подготовить КП для ООО «Ромашка» — 3 дня (@Иванов)
   2. Ответить клиенту по рекламации — 2 дня (@Петров)
   ...
   
   🟡 Сделки без движения (3):
   1. ООО «Вектор» — 8 дней без активности (450 000 ₽)
   ...
   
   ✅ Задачи без дедлайна: 12
   
   [Открыть подробности]
   ```
2. **Настройки приложения**: форма — время отправки (09:00), дни (пн–пт), получатели (чекбоксы), что включать (задачи ✓, сделки ✓, без дедлайна ✓), порог дней без активности (7).
3. **Выбор получателей**: интерфейс — «Отправлять в: ○ Личный чат ○ Групповой чат [выбрать]». Список сотрудников с чекбоксами.
4. **Полный отчёт (по ссылке из дайджеста)**: развёрнутый экран — все просроченные задачи + все сделки без движения, с возможностью перехода в карточку.
5. **Пустой дайджест**: «✅ Всё под контролем! Просрочек нет, все задачи активны. Хорошего дня!» (если включена опция «отправлять даже если пусто»). По умолчанию — не отправляется.
6. **История дайджестов**: список отправленных дайджестов с датами — «28.05 — 5 просрочек, 3 сделки», «27.05 — 7 просрочек, 2 сделки» (тренд).
7. **Мобильное уведомление**: push-нотификация на телефоне с превью дайджеста.

## Стратегия минимальных прав

| Право | Обоснование | Уровень |
|-------|-------------|---------|
| task.item.read | Определение просроченных и зависших задач | Чтение |
| crm.deal.read | Определение сделок без активности | Чтение |
| crm.activity.read | Дата последней активности по сделке | Чтение |
| im.message.add | Отправка дайджеста в чат | Запись (только сообщения) |
| user.basic | Имена ответственных для дайджеста | Чтение |

**Не запрашиваем:**
- Запись/изменение задач или CRM
- Полный доступ к чатам (чтение чужих сообщений)
- Доступ к контактам/компаниям
- Доступ к файлам, диску, календарю
- Доступ к телефонии/записям звонков

## Что приложение НЕ читает / НЕ хранит

- ❌ Содержание сообщений в чатах
- ❌ Описания и комментарии задач (только заголовок)
- ❌ Персональные данные клиентов
- ❌ Сумму/содержимое сделок (только название + дата последней активности)
- ❌ Записи звонков, файлы, вложения

## Политика хранения

- Храним: настройки расписания + история отправок (дата, кол-во элементов в дайджесте)
- Кэш данных: формируется за 5 мин до отправки, живёт 1 час
- Не храним: содержимое задач, сделок, переписок
- Срок истории отправок: 30 дней
- При удалении: все данные удаляются, дайджесты прекращаются

## Производительность

- Запрос данных: 1 раз в день (за 5 мин до отправки)
- API-вызовы: 2–3 запроса (просроченные задачи + сделки без активности)
- Формирование: < 500 мс (агрегация + форматирование)
- Отправка: 1 сообщение в чат (< 100 мс)
- Нулевая нагрузка между отправками (нет фоновых процессов)

## FAQ (текст для Маркета)

**Вопрос: Чем отличается от «Ежедневного отчёта» (PALAX)?**
Ответ: PALAX — статистика (сколько звонков, задач закрыто). Мы — проблемы (что просрочено, где нет активности). Не «что было», а «что требует действия».

**Вопрос: Приложение читает мои чаты?**
Ответ: Нет. Приложение только ОТПРАВЛЯЕТ сообщения (im.message.add). Оно не имеет доступа к чтению чужих сообщений или истории чатов.

**Вопрос: Что если каждый день 100 просрочек?**
Ответ: Дайджест показывает топ-5 по критичности + «и ещё 95 →» со ссылкой на полный список. Не перегружаем чат.

**Вопрос: Можно ли получать дайджест реже?**
Ответ: Да. Варианты: ежедневно (пн–пт), 3 раза в неделю (пн/ср/пт), еженедельно (пн). Выбирайте в настройках.

**Вопрос: Если всё хорошо — придёт пустое сообщение?**
Ответ: Нет. По умолчанию, если проблем нет — дайджест не отправляется. Можно включить «отправлять всегда» для подтверждения, что всё ок.

**Вопрос: Какие данные хранятся?**
Ответ: Только настройки и история отправок (дата + кол-во элементов). Не храним содержимое задач, сделок или чатов.
