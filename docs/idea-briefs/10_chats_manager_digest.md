# 10 — Дайджест для руководителя

## Подготовка (Шаг 0)

**Для руководителей отделов и проектов, которые тратят 20–40 минут каждое утро на просмотр задач, сделок и просрочек в Bitrix24 — приложение автоматически формирует краткий дайджест «что горит / что просрочено / где нужно решение» и отправляет его в чат, экономя 2+ часа в неделю.**

## Идея

| Поле | Значение |
|------|----------|
| ICP/роль покупателя | Руководитель отдела, тимлид, РОП (компании 10–300 чел.) |
| Боль | Каждое утро нужно вручную проверять статус задач, сделок, просрочек. Информация разбросана — легко что-то пропустить. |
| Измеримый эффект | Экономия 20–40 мин/день; 0 пропущенных критических событий (просрочки, зависания) |
| MVP-объём | 1 неделя: ежедневный отчёт в чат (просрочено, горит, ждёт решения) + настройка расписания и получателей |
| Основные риски | Шум (слишком длинный дайджест при плохой дисциплине); конкуренция с приложением PALAX |

## Скоринг

| Критерий | Оценка (1–5) | Комментарий |
|----------|:---:|-------------|
| A1 — частота боли | 5 | Ежедневная рутина руководителя |
| A2 — платёжность/ROI | 4 | Экономия времени руководителя — ощутимый ROI |
| A3 — дифференциация | 3 | Есть конкурент (PALAX), но он фокусируется на статистике, а не на «что горит» |
| B1 — стандартные объекты | 5 | Задачи + сделки — стандартные |
| B2 — понятный источник | 5 | Просроченные задачи + сделки без активности — чёткие критерии |
| B3 — минимальные права | 3 | Чтение задач + сделок + отправка сообщений в чат |
| B4 — без сложной инфраструктуры | 5 | Cron + запросы к API + сообщение в чат |
| C1 — один killer-экран | 5 | Настройка дайджеста (что включать, когда отправлять) |
| C2 — минимум фоновых процессов | 5 | Один запуск в день по расписанию |
| C3 — производительность | 4 | Агрегация за текущий день — нетяжёлая |
| C4 — управляемые edge-cases | 4 | Пустой дайджест → не отправлять; длинный → топ-10 |
| **Итого** | **4.4** | Высокий приоритет |

## Красные флаги — НЕТ

- Нет realtime (1 запуск/день)
- Ясная ценность
- Минимальная инфраструктура
- Time-to-value: установил → настроил расписание → утром получил дайджест
