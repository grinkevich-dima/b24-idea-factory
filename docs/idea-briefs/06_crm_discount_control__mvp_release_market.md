# 06 — Скидки под контролем: MVP + релиз + текст Маркета

## УТП

**Единственное приложение на Marketplace, которое не применяет скидки, а контролирует их — автоматические пороги, журнал нарушений и мгновенный алерт руководителю при превышении.**

## MVP (3–5 фич)

1. **Правила порогов скидок** — настройка максимальной скидки (%) по воронке, стадии или роли менеджера.
2. **Детектор нарушений** — при сохранении сделки с товарами проверяет скидку каждой позиции vs порог.
3. **Журнал отклонений** — таблица: дата, менеджер, сделка, товар, порог, факт, превышение.
4. **Алерт руководителю** — уведомление в чат при каждом нарушении (с ссылкой на сделку).
5. **Настройки** — экран управления правилами + вкл/выкл уведомлений.

## Не-MVP (отложено)

- Блокировка сохранения сделки при превышении (требует другого подхода к UX)
- Накопительные/ступенчатые скидки (слишком вариативно для MVP)
- Интеграция с прайс-листами
- Аналитические отчёты по потерям маржи
- Роль «аппрувер скидки» (workflow согласования)

## План релиза

### Неделя 1
- День 1–2: Архитектура + обработчик события изменения сделки
- День 3–4: Логика проверки скидок vs пороги + запись в журнал
- День 5: UI настройки правил (одна страница)

### Неделя 2
- День 1–2: UI журнала нарушений (таблица + фильтры)
- День 3: Отправка уведомлений в чат
- День 4: Тестирование на тестовом портале
- День 5: Подготовка к публикации + тексты Маркета

## Риски и смягчения

| Риск | Смягчение |
|------|-----------|
| Разные модели скидок | MVP: только % скидка на товар. Суммовую — в v2 |
| Сопротивление менеджеров | Позиционирование: «защита маржи для руководителя», а не «слежка» |
| Высокая нагрузка при массовом изменении | Debounce + проверка только при смене суммы/скидки |

## Черновик текста для Маркета

### Заголовок
**Скидки под контролем — защита маржи в CRM**

### Подзаголовок
Автоматические пороги скидок, журнал нарушений и мгновенный алерт руководителю. Больше никаких «скидок от себя».

### Буллеты ценности
- ✅ Настройте максимальную скидку по воронке/стадии/роли — за 2 минуты
- ✅ Мгновенное уведомление при превышении порога
- ✅ Полный журнал: кто, когда, на сколько превысил
- ✅ Минимальные права — только чтение товаров + уведомления
- ✅ Работает сразу после установки — без долгой настройки

### Quick Start
1. Установите приложение
2. Задайте порог скидки (например, 15%) для нужной воронки
3. Укажите получателя уведомлений (РОП / коммерческий директор)
4. Готово — каждое превышение фиксируется и алертится

### FAQ
- **Блокирует ли приложение сохранение сделки?** — Нет, в MVP только фиксация и алерт. Менеджер может сохранить, но руководитель узнает мгновенно.
- **Можно ли задать разные пороги для разных менеджеров?** — Да, правила настраиваются по роли/должности.
- **Какие права нужны?** — Чтение товарных позиций сделки + отправка уведомлений. Без полного доступа к CRM.
