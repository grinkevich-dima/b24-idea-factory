# 09 — KPI РОПа: одна панель — скриншоты + безопасность + FAQ

## План скриншотов (8)

1. **Главный экран — Панель метрик**: 12 карточек в сетке 3×4. Каждая: число + название + Δ% (зелёный/красный). Пример: «Новые сделки: 14 ↑+27%», «Конверсия: 32% ↓-5%».
2. **Фильтр по менеджеру**: выпадающий список менеджеров → панель показывает метрики конкретного сотрудника.
3. **Фильтр по периоду**: переключатель «Сегодня / Вчера / Неделя / Месяц» — числа мгновенно обновляются.
4. **Детализация метрики**: клик на «Просроченные дела: 7» → список дел с датами, ответственными, привязанными сделками.
5. **Аномалия (алерт)**: красная карточка с пульсирующим значком: «⚠️ Конверсия упала на 35% vs прошлая неделя». Кнопка «Подробнее».
6. **Уведомление в чате**: «📊 Аномалия KPI: конверсия лид→сделка упала с 28% до 18% (−35%). Проверьте воронку. [Открыть панель]».
7. **Мобильный вид**: адаптивная версия — карточки в 1 колонку, свайп для прокрутки.
8. **Пустое состояние (нет данных)**: «За выбранный период нет данных. Попробуйте расширить период.»

## Стратегия минимальных прав

| Право | Обоснование | Уровень |
|-------|-------------|---------|
| crm.lead.read | Подсчёт новых лидов, конверсия | Чтение |
| crm.deal.read | Сделки: количество, суммы, конверсия | Чтение |
| crm.activity.read | Просроченные дела, звонки | Чтение |
| im.message.add | Алерт при аномалии | Запись (сообщения) |
| user.basic | Список менеджеров для фильтра | Чтение |

**Не запрашиваем:**
- Запись/изменение CRM-сущностей
- Доступ к контактам/компаниям (не нужны для метрик)
- Полный доступ к чатам
- Доступ к задачам, диску, календарю

## Что приложение НЕ читает / НЕ хранит

- ❌ Персональные данные клиентов (ФИО, телефоны, email)
- ❌ Содержимое переписок и комментариев
- ❌ Финансовые документы и счета
- ❌ Записи звонков
- ❌ Данные за пределами CRM (задачи, чаты, диск)

## Политика хранения

- Храним: только агрегированные числа (количества, суммы, конверсии) за период
- Кэш: обновляется каждые 15 минут, хранится 24 часа
- История аномалий: 30 дней (для отслеживания трендов)
- Не храним: данные отдельных сделок, лидов, контактов
- При удалении: все данные (кэш + настройки) удаляются

## Производительность

- Кэш агрегатов: обновление каждые 15 мин (не при каждом открытии)
- Запросы: batch API (лиды + сделки + дела за период) — 1–3 запроса
- Первая загрузка: < 2 секунды для порталов до 5000 сделок/мес
- Большие порталы: инкрементальное обновление (Δ с последнего кэша)
- UI: статические числа из кэша, без тяжёлых графиков

## FAQ (текст для Маркета)

**Вопрос: Чем отличается от встроенной CRM-аналитики?**
Ответ: Одна страница vs 10 разделов. Фокус на «сегодня» и «вчера», а не месячные отчёты. Лиды + сделки + дела + звонки — всё в одном месте. Алерт при аномалии.

**Вопрос: Нужно ли настраивать планы продаж?**
Ответ: Нет. Работает сразу — показывает факт + сравнение с прошлым периодом. Планы (plan/fact) — в будущей версии.

**Вопрос: Поддерживает ли несколько воронок?**
Ответ: Да. Фильтр по воронке или «все воронки». Метрики пересчитываются мгновенно.

**Вопрос: Приложение замедляет портал?**
Ответ: Нет. Данные кэшируются. Обновление — раз в 15 мин в фоне. Открытие панели не создаёт API-нагрузки.

**Вопрос: Можно ли видеть метрики конкретного менеджера?**
Ответ: Да. Фильтр по менеджеру — все 12 метрик пересчитываются для выбранного сотрудника.

**Вопрос: Какие данные хранятся?**
Ответ: Только агрегаты (количества, суммы, проценты). Не храним данные отдельных сделок, контактов или переписок. При удалении приложения — всё удаляется.
