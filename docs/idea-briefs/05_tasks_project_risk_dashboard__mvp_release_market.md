# Идея 05 — Задачи: “Риск проекта за 1 экран” — UTP, MVP, релиз, Маркет

## 1) УТП
**Один экран “где горит” по проекту**: риск = понятные сигналы (просрочки, без срока, перегруз, блокеры) + алерт при росте риска.

## 2) MVP (3–5 фич)
1. **Выбор проекта/группы задач** + сводка риска (низкий/средний/высокий).
2. **4 ключевых сигнала**:\n
   - просроченные;\n
   - без дедлайна;\n
   - “нет изменений N дней”;\n
   - перегруз исполнителя (WIP > порога).
3. **Топ‑список причин**: какие задачи/исполнители дают максимум риска.
4. **Настройки порогов** (простые): N, %, WIP.
5. **Алерт при росте риска** (в чат/уведомление).

## 3) Формула риска (прозрачно)
RiskScore = w1*OverdueRate + w2*NoDeadlineRate + w3*StaleRate + w4*WIPOverloadRate\n
В MVP веса фиксированные + 2 пресета (“строго/мягко”).

## 4) План релиза 1–2 недели
### Неделя 1
- Сбор метрик по задачам в выбранном проекте.
- Экран риска + топ сигналов + детализация.

### Неделя 2
- Настройки порогов + пресеты.
- Уведомления при росте риска + дайджест.
- Оптимизация (кэш агрегатов по проекту).

## 5) Метрики успеха
- Снижение доли просроченных задач по проектам.
- Сокращение “stale задач” (>N дней без изменений).
- Среднее время реакции после алерта.

## 6) Риски
- “Риск” должен быть объяснимым → только rule‑based, без “чёрного ящика”.
- Разные процессы → пресеты + настройка порогов.

## 7) Текст для Маркета (черновик)
### Заголовок
Риск проекта по задачам: дашборд и сигналы

### Подзаголовок
Показывает зоны риска по задачам и предупреждает, когда проект начинает “гореть”.

### Что внутри
- Риск‑панель по проекту
- Просрочки/без срока/зависшие/перегруз
- Настраиваемые пороги и алерты
