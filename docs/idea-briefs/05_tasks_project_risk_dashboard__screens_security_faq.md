# Идея 05 — Задачи: “Риск проекта” — скрины, безопасность, FAQ

## 1) План скриншотов (6–8)
1. **Главный экран риска проекта**\n
   - RiskScore (низкий/средний/высокий) + объяснение “почему”.\n
   - фильтр: проект/группа, период.
2. **Сигналы риска (4 карточки)**\n
   - просроченные\n
   - без дедлайна\n
   - без изменений N дней\n
   - перегруз (WIP).
3. **Топ задач‑виновников**\n
   - список задач, которые дают максимум риска.\n
   - переход в задачу.
4. **Топ исполнителей по риску**\n
   - у кого просрочки/перегруз, сколько.\n
5. **Настройки порогов и пресеты**\n
   - “строго/мягко”, N дней “stale”, WIP‑порог.\n
6. **Алерты**\n
   - пример уведомления “риск вырос до высокого”.

## 2) Чек‑лист модерации и безопасности
### Права
- **Задачи**: чтение задач в выбранных проектах/группах.\n
- **Чат и уведомления**: отправка алертов (без чтения сообщений).\n
- **Пользователи (минимальный)**: отображение исполнителей.

### Хранение
- Только агрегаты и timestamps (дедлайны, статус просрочки, “последнее изменение”), без содержимого.

### Производительность
- Считать только выбранный проект/группу и активные задачи.\n
- Кэшировать агрегаты по проекту.

## 3) FAQ (готовый текст)
**Что такое риск проекта?**\n
Это прозрачная, rule‑based оценка на основе 4 сигналов: просрочки, задачи без срока, “зависшие” задачи и перегруз исполнителей.

**Это BI‑отчёт?**\n
Нет. Это отдельный простой дашборд “один экран”, без обязательной зависимости от BI‑конструктора.

**Можно ли настроить пороги?**\n
Да, есть пресеты и ручная настройка.
